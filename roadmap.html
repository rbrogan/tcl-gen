Roadmap

We hope to do a series of small releases (hopefully one every two weeks). Each release would consist of both a modest installment of commands as well as project improvements. Our main goal here from a project management POV is to keep things moving with commitments to hard dates and have those drive project improvements. The main improvements we may look to fit in at this point are:

  1. Partial Loading of Gen: This feature will check the user machine for packages and versions and load those commands which the user can support. Note that, this is not strictly necessary since the user can simply run the test suite and find out themselves. Even if a command is not supported, it can still be loaded and go unused. However, this feature would help build infrastructure for the Gen^2 project.
  2. Support for TEPAM / Support for flags: There are a number of features from TEPAM we want, especially support for flags. For many commands it is awkward to go without flags. We also may want TEPAM support for validation / error reporting since for many commands that is taking up a fair amount of code.
  3. Overhaul of Error Conditions: This is mainly because we find that section of the documentation does look very good, but it is the type of problem we must sit down and analyze and figure out what is the best way to go.
  4. Exception Policies: This also sticks out when writing many commands. Should we silently change an index out-of-bounds to be the end index (as some commands do)? Should we raise an error? At this point, the choice feels too arbitrary. We would rather have a basic set of default policies and give the user the ability to override and make them be consistent with their own policies.
